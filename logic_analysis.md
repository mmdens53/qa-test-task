# Анализ логики функции checkAccess

## Назначение функции
Функция определяет, может ли пользователь получить доступ к мероприятию
на основе возраста, наличия билета или приглашения.

## Найденные проблемы

1. Несоответствие условия и сообщения об ошибке  
   Условие:

```js
hasTicket == false && invited == false

Сообщение:
> "Нет билета или приглашения"

Логически корректнее: **"Нет билета и приглашения"**.  
Текущее сообщение может вводить пользователя в заблуждение.

2. Отсутствие обработки некорректных входных данных  
Типы параметров не проверяются, что может привести к
некорректному поведению функции.

## Рекомендации
- Привести текст сообщений в соответствие с логикой условий.
- Добавить валидацию входных параметров.
- Использовать более явную структуру условий для повышения читаемости.
