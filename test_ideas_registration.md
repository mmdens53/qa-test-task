# Идеи тестирования формы регистрации

## Позитивные сценарии
- Регистрация с валидными данными во всех обязательных полях.
- Регистрация с минимально допустимой длиной пароля.
- Регистрация с максимально допустимой длиной пароля.

## Негативные сценарии
- Регистрация с уже существующим E-mail.
- Регистрация без заполнения обязательных полей.
- Несовпадение пароля и подтверждения пароля.
- Ввод некорректного E-mail (без `@`).
- Ввод пробелов или невидимых символов в поля.

## Проверки безопасности
- Попытка SQL-инъекции в текстовых полях.
- Попытка XSS-инъекции.

## Приоритетные проверки
1. Успешная регистрация с валидными данными (базовый позитивный сценарий).
2. Регистрация с уже существующим E-mail (предотвращение дубликатов).
3. Регистрация без обязательных полей (валидация формы).
