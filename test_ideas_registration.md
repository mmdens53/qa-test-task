# Идеи тестирования формы регистрации

## Полный набор идей

1. Регистрация с валидными данными во всех обязательных полях. Ожидаемый результат: регистрация успешна.
2. Повторная регистрация с уже использованным E-mail. Ожидаемый результат: система блокирует создание дубликата, отображает сообщение об ошибке.
3. Регистрация без заполнения всех обязательных полей. Ожидаемый результат: отображаются сообщения о необходимости заполнения полей.
4. Проверка минимальной и максимальной длины пароля (например, 4 и 65 символов). Ожидаемый результат: система корректно обрабатывает граничные значения согласно требованиям.
5. Ввод несовпадающих пароля и подтверждения пароля. Ожидаемый результат: сообщение об ошибке, регистрация не выполняется.
6. Поочередное оставление каждого из обязательных полей пустым. Ожидаемый результат: корректные сообщения о валидации для каждого поля.
7. Ввод некорректного E-mail (например, без символа «@»). Ожидаемый результат: ошибка валидации E-mail.
8. Ввод только пробелов или невидимых символов в полях. Ожидаемый результат: такие значения считаются пустыми, отображается ошибка.
9. Проверка устойчивости к SQL-инъекциям и XSS-инъекциям в текстовых полях. Ожидаемый результат: данные корректно экранируются/отклоняются, ошибок серверной части нет.


## Три первоочередные проверки

1. Регистрация с валидными значениями во всех обязательных полях — базовая позитивная проверка успешной регистрации.
2. Регистрация с уже существующим E-mail — проверка предотвращения создания дубликатов учетных записей.
3. Попытка регистрации без заполнения всех обязательных полей — проверка обязательности полей и сообщений валидации.
